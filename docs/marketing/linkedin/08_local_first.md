# Local-first design for developer operations

## Body

Developer workflows contain sensitive context: local paths, task history, logs, credentials, and source code.

AI Workstation is designed around local collection and local rendering. The core status dashboard does not require uploading the development environment to a new hosted service. Optional integrations are configured only when the user chooses to use them.

Local-first design creates deployment work, but it also gives customers a clearer data boundary. For a personal or team status dashboard, that tradeoff can be appropriate.

The commercial service exists partly to absorb the setup complexity without changing the privacy model.
