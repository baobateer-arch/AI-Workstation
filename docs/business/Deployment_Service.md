# AI Workstation｜部署服务说明

## 服务目标

把“开源项目可以运行”转化为“客户在自己的设备上可以稳定使用”。部署服务关注实际交付，不要求客户先理解项目结构。

## Personal Setup 交付流程

1. **适配确认**：确认 Windows 环境、Kindle 型号、已使用 Agent 和期望展示内容。
2. **安装准备**：说明需要客户自行准备的设备、账号和第三方订阅。
3. **本地安装**：完成 AI Workstation 基础安装与运行检查。
4. **Agent 配置**：配置客户已使用的 Claude Code、Codex 或 MiMo 环境。
5. **Kindle 设置**：完成仪表盘显示与刷新检查。
6. **验收说明**：确认主要状态可见，并说明日常启动和常见问题处理方式。

## 客户需要准备

- 可用的 Windows 电脑。
- 计划使用的 Kindle 设备及型号信息。
- 已安装或计划使用的 AI Agent。
- 可以配合完成本地操作的时间。

请勿通过聊天或邮件直接发送 API Key、Cookie、Token、私有代码或敏感日志。

## 验收标准

- AI Workstation 可以启动并生成仪表盘。
- 约定范围内的 Agent 信息可以显示。
- Kindle 可以按约定方式看到仪表盘。
- 客户知道如何启动、查看和反馈问题。
