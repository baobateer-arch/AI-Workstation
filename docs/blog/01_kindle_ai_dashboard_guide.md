# Kindle AI Dashboard：如何把旧 Kindle 变成 AI 工作状态屏

> SEO description：了解 Kindle AI Dashboard 的适用场景、核心设计、设备价值和部署方式，把闲置 Kindle 变成低功耗 AI Agent 控制中心。

很多人的 Kindle 在读完几本书后就长期闲置。但电子墨水屏拥有一个普通副屏很难替代的优势：低功耗、无背光干扰、静态内容始终可见。对于需要同时运行多个 AI Agent 的开发者，它可以成为一块非常合适的状态屏。

## Kindle AI Dashboard 是什么

Kindle AI Dashboard 不是把终端完整搬到电子书上，而是提取最重要的工作信号。例如：Claude Code 是否正在运行、Codex 是否等待授权、MiMo 是否停止，以及 Codex quota 或模型余额是否接近限制。

AI Workstation 将这些信息整理成高对比仪表盘，再交给 Kindle 持续显示。主显示器负责写代码和沟通，Kindle 只负责回答一个问题：现在是否需要你介入？

## 为什么 Kindle 适合这个场景

第一，状态信息通常变化不快，不需要高刷新率。第二，电子墨水屏不会像普通副屏一样不断吸引注意力。第三，闲置设备可以获得一个清晰、长期的用途。

这种设计也迫使产品控制信息密度。真正有价值的内容会留下，日志、动效和装饰性指标则留在主机上。

## 适合哪些用户

如果你只偶尔使用单个 Agent，普通终端可能已经足够。如果你经常让两个或更多 Agent 并行工作、运行长任务，或容易错过授权提示，持续可见的状态屏会更有价值。

## 如何开始

你可以自行使用开源项目安装，也可以选择 Personal Setup，由服务方完成 AI Workstation、Kindle dashboard 和 Agent 环境的基础配置。开始前应先确认 Kindle 型号、Windows 环境和计划接入的 Agent。
