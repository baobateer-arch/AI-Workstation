# 多 AI Agent 工作站如何设计：状态、分工与人工介入

> SEO description：介绍多 Agent 工作站的设计方法，包括 Agent 分工、状态模型、审批节点和仪表盘优先级。

同时打开多个 Agent 不等于拥有多 Agent 工作流。真正稳定的系统需要知道每个 Agent 负责什么、何时需要人、出现异常后如何处理。

## 从分工开始

可以按任务类型分配 Agent，例如实现、审查、文档或实验。但分工必须结合团队实际工具和验证方式，不能只根据模型标签决定。

## 统一状态语言

不同工具的日志格式可能完全不同。统一为少量用户可理解的状态，能够让仪表盘保持一致。RUNNING 表示工作进行中，IDLE 表示暂时无活动，WAITING_APPROVAL 表示需要人，STOPPED 表示应检查结果或重新启动。

## 把人工介入设计进去

审批不是失败，而是工作流的一部分。系统应清楚告诉用户为什么需要介入，并让这个状态在合适的设备上持续可见。

## 控制信息密度

多 Agent 仪表盘最容易犯的错误是显示过多细节。展示层应优先支持决策，完整日志留在原工具中。Kindle 等电子墨水屏尤其适合这种分层。

Pro AI Workflow 服务会基于具体场景梳理分工、自动化和定制仪表盘，而不是套用一张固定模板。
